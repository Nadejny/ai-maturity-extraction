### Три кластера поведения

**Кластер A — «нормальные»** (Qwen family + llama31): 46–50% mat=0, богатое распределение по mat=2.

**Кластер B — under-detectors** (gemma4, mistral): 70–73% mat=0. Слишком консервативны, массово пропускают AI.

**Кластер C — особенность mistral:** уникально активно использует **mat=1=pilot** (147 документов, 19%). Все остальные модели ставят mat=1 крайне редко (7–21 документ). Это не качество — это систематическая путаница между «production» и «pilot».

### Особенности по размеру Qwen

- mat=0 ~ стабильно (49% / 50% / 48.5%)

### Особенность llama31

llama31 даёт **386 документов на mat=2 (50.2%)** — больше, чем любая другая модель. Но **почти никогда** не ставит mat=3 (всего 7 docs). Она «видит AI», но не отличает single-function от multi-function. Для финальной аналитики по уровню зрелости llama31 плохо подходит.

## 4. Parse error и wrong_field_names

На 769 документах стабильно наблюдается:

| Модель | Parse errors | Wrong fields | Alias used | Combined schema issues |
|--------|--------------|--------------|-------------|-----------------------|
| llama31_8b | **3 (0.39%)** | **2 (0.3%)** | 86 | 91 (11.8%) |
| mistral_7b | 3 (0.39%) | 11 (1.4%) | 76 | 90 (11.7%) |
| qwen3_14b | 10 (1.30%) | **7 (0.9%)** | 3 | 20 (2.6%) |
| qwen3_8b | 11 (1.43%) | 20 (2.6%) | 4 | 35 (4.6%) |
| qwen3_32b | 25 (**3.25%**) | 9 (1.2%) | 8 | 42 (5.5%) |
| gemma4_e4b | 13 (1.69%) | **43 (5.6%)** | 7 | 63 (8.2%) |

**Два типа проблем:**
1. **Parse error** (невалидный JSON, обычно repetition loop) — больше у крупных моделей (32b=25).
2. **Schema adherence** (wrong field names + alias used) — больше у маленьких моделей.

**Поразительный факт про llama31:** только 2 wrong field names, но 86 alias used. То есть llama31 стабильно использует `"ai_spans"` вместо `"evidence_spans"` — выдаёт это вроде всегда, что лишь формально «alias», но фактически модель не освоила правильное имя поля. У mistral тот же паттерн (76 alias). Это специфика instruction-tuned моделей не-Qwen/Gemma происхождения.

**Qwen3_14b — чемпион по комбинированной схемной чистоте:** всего 20 schema issues (2.6%), при том что llama31 формально выше (только 3 parse errors + 2 wrong fields, но 86 alias инфекции).

**Qwen3_32b — самый нестабильный по парсингу:** 3.25% parse errors. Причина — длинные ответы (838 completion tokens vs 329 у llama31 — ×2.5) → больше возможностей для repetition loop.

### Оценка 17.5 часов — подтвердилась

Исходный прогноз 16.9 часов на qwen32 (из 79.1s × 769) оказался точным: фактически **17.46 часа** (81.7s × 769). Расхождение 3.3% — техническое, связано с тем, что qwen32 немного замедлился на длинных документах.

## 14. Рекомендация для диплома

### Primary model для финального анализа: **qwen3_32b**

Обоснование:
- Максимальная разрешающая способность между mat=2 и mat=3 (в 2× больше multi-function docs, чем у 8b/14b).
- Единственная модель с useful extraction для risk_signals (35%) и org_change (25%).
- Самые богатые evidence spans (8.0/doc) → наилучшая поддержка каждого заявленного признака.
- Rationale contrastive, fact-grounded (упоминает конкретные KPIs, команды, продукты).

Ограничения, которые нужно упомянуть в тексте диплома:
- 3.25% parse errors — самый высокий у 32b. Это ~25 документов, которые будут исключены из анализа.
- 17.5 часов время — не позволяет быстро переиспытывать promts.

### Sensitivity analysis: **qwen3_14b**

Параллельно с qwen32b:
- Confirmation that качественные выводы устойчивы к выбору модели.
- Опубликовать inter-model variance как «доверительный коридор» для каждой отраслевой метрики.
- qwen3_14b — второй кандидат благодаря 86.6% согласию с 8b и 83.9% с 32b.

### Архитектурный контр-пример: **gemma4 + mistral**

В тексте диплома показать, что:
- 70%+ mat=0 у gemma/mistral — артефакт плохой instruction-following, **не** свойство корпуса.
- Сравнение с qwen/llama даёт явное **архитектурное превосходство Qwen3 и Llama3.1** при 8B масштабе.
